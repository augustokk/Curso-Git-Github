Criando seu primeiro projeto

Na sua pagina do GitHub clique em Projects

Nessa pagina voce eh capaz de escolher se quer usar um dos templates mais usados (Featured) ou comercar do inicio escolhendo se prefere Table, Board, ou Roadmap

Apos selecionar isso voce ira colocar o nome do seu projeto e entao clicar em create project

Dentro do project settings voce eh capaz de colocar uma breve descricao e tambem adicionar um README, alem de opcoes como fechar o projeto/ deletar ele ou mudar visibilidade de privado para publico


Dentro do Projeto


Mudando o layout das Views

Na pagina dos seus projetos, voce eh capaz de criar inumeras paginas diferentes de view e dentro delas pode usar em cada uma um approach diferente, por exemplo Table, Board ou Roadmap


Table layout

A tabela é uma planilha poderosa e flexível que inclui seus problemas, pull requests e rascunhos com metadados do GitHub e campos personalizados do seu projeto. Você pode agrupar, classificar e filtrar itens, além de mostrar ou ocultar campos na tabela para atender às necessidades da sua equipe.


Board layout

O layout do quadro distribui seus problemas, pull requests e rascunhos em colunas personalizáveis. Você pode criar um quadro kanban definindo o campo da coluna como um campo de 'Status' ou qualquer outro campo de seleção única ou de iteração. Você pode arrastar itens individuais ou múltiplos de uma coluna para outra, e o valor desses itens será ajustado para corresponder à coluna para a qual você os arrastou.


Roadmap layout

O layout do roadmap oferece uma visualização de alto nível do seu projeto ao longo de um período configurável, permitindo arrastar itens para ajustar suas datas de início e término ou a iteração selecionada. Os roadmaps usam seus campos de data e iteração personalizados para posicionar seus problemas, pull requests e rascunhos em uma linha do tempo, permitindo acompanhar o trabalho ao longo do tempo e observar o progresso. Você também pode exibir linhas verticais para destacar datas importantes do seu projeto, incluindo iterações, marcos e datas de itens no seu projeto. Esses marcadores ajudam a obter uma visão clara da sua carga de trabalho futura e como ela está distribuída entre iterações ou marcos


Usando Issues

Participar de projetos atraves de issues

O ideal para quem quer comecar a trabalhar em projetos eh comecar a utilizar issues em diferentes projetos

Basicamente o plano eh entrar em um repositorio e ver se existe algum problema, voce pode ajuda apenas dizendo o que aconteceu por exemplo, ou caso voce tenha conhecimento, voce poda clonar o repositorio, analisar o codigo e tentar resolver, se conseguir resolver, cria um fork, cria uma branch e abre um pull request

Caso o dono do projeto entenda que eh valido essa sua solucao pro codigo, ele vai aceitar esse request e voce se torna colaborador do projeto

Nao crie uma issue em um projeto sem antes ver se ja existe uma issue pra isso

Voce eh capaz de adicionar comentarios e reacoes nos issues

Quando alguem cria um issue no seu repositorio, voce pode adicionar um comentario nele, pode fechar o issue e pode tambem trancar o issue usando o lock conversation, dessa forma ninguem mais pode adicionar detalhes ao issue, mas ainda assim podem ver


Assinatura padrao / Saved replies

clicando no seu perfil, depois clicando em settings, na coluna da esquerda voce ira achar a aopcao saved replies, voce pode criar uma assinatura, uma resposta padrao ou algum reply que voce constantemente ira usar, pois assim voce ira conseguir, usando esse atalho, responder muito mais rapido

para usar o saved reply eh muito facil e pratico, na hora que voce for comentar, existe uma seta no camppo alto esquerdo na caixa de comentario, que voce clicando nela, aparecem as opcoes de saved reply
